## ğŸ›ï¸ Relational vs Non-relational Databases
### Relational Databases (SQL)

- Store data in tables (rows + columns)

- Use SQL

- Have a fixed schema

- Great for structured data and complex joins

### Examples: `MySQL`, `PostgreSQL`,`SQL Server`

- Non-relational Databases (NoSQL)

- Store data as documents, key-value pairs, graphs

- Schema is flexible

- Great for big, unstructured, or rapidly changing data

### Examples: `MongoDB`, `Firebase`, `Cassandra`

## Quick Comparison
| Feature   | Relational (SQL)  | Non-Relational (NoSQL) |
| --------- | ----------------- | ---------------------- |
| Structure | Tables            | Documents / Key-Value  |
| Language  | SQL               | Varies                 |
| Schema    | Fixed             | Flexible               |
| Examples  | MySQL, PostgreSQL | MongoDB, Firebase      |
